What are Regular Expressions?

Regular expressions (Regex) are a way of describing patterns in a string of data, which allows you to search for data strings, like email addresses or passwords, that match that pattern

Creating regular expressions in js

To create regular expression, use one of the ways:

With the help of literals / ... / - designations of regular expressions.Example:

let reg = /Hello /;

Through a special REGEXP:

let reg = New RegExp('Hello');

Basic Syntax

A regular expression is typically composed of:

  • Literal characters: Represent exact matches
  • Metacharacters: Special characters with predefined meanings
  • Quantifiers: Specify the number of occurrences
  • Anchors: Define positions within the text

Literal Characters

Literal characters in regex represent themselves and match exactly the same character in the input text.

However, special characters like +, *, and $ have special meanings and need to be escaped with a backslash to match literally.

For example, the regex /hello/ matches the string "hello" in the input text.

Metacharacters

Metacharacters are special characters in regex that have predefined meanings and functions.Some common metacharacters include:

  • . - Find a single character, except newline or line terminator
  • \w - Find a word character
  • \W - Find a non-word character
  • \d - Find a digit
  • \D - Find a non-digit character
  • \s - Find a whitespace character
  • \S - Find a non-whitespace character
  • \b - Find a match at the beginning/end of a word, beginning like this: \bHI, end like this: HI\b
  • \B - Find a match, but not at the beginning/end of a word
  • \0 - Find a NULL character
  • \n - Find a new line character
  • \f - Find a form feed character
  • \r - Find a carriage return character
  • \t - Find a tab character
  • \v - Find a vertical tab character
  • \xxx - Find the character specified by an octal number xxx
  • \xdd - Find the character specified by a hexadecimal number dd
  • \udddd - Find the Unicode character specified by a hexadecimal number dddd

Examples


		let reg = /m.d/g;
    let str = "He's mad!"; //matches

    let reg = /o+/g;
    let str = "Codeee!"; //matches

    let reg = /50*/g;
    let str = "5, 500 or 5000"; //matches

    let reg = /code(?= Alex)/g;
    let str = "code with Alex"; //matches

Quantifiers

Quantifiers specify the number of occurrences of a character or group in regex.

  • *: Matches zero or more occurrences
  • +: Matches one or more occurrences
  • ?: Matches zero or one occurrence
  • {n}: Matches exactly n occurrences
  • {n,}: Matches at least n occurrences
  • {n,m}: Matches between n and m occurrences

Examples

Exact Count {n}


let str = 'Year: 2022';
let re = /\d{4}/; // Matches a four-digit number basically
                // concise & better way to write \d\d\d\d
console.log(str.match(re)); // Output: ["2022"]

The Range {n,m}


let str = "The meeting is scheduled for 10:30 AM and ends at 2 PM";
let re = /\d{2,4}/g; // Matches numbers with 2 to 4 digits
console.log(str.match(re)); // Output: [ '10', '30' ]

Examples

{n,}


    let str = 'The price of the item is $2500';
    let re = /\d{2,}/g; // Matches numbers with 2 or more digits
    console.log(str.match(re)); // Output: ["2500"]

Shorthands: +, ?, *


    let phone = "+48-(886)-777-010";
    let result = phone.match(/\d+/g); // Matches one or more digits
    console.log(result); // Output: ["48", "886", "777", "010"]

Examples

Zero or One (?)


let str = 'The sky is blue in color, the ocean is blue in colour';
console.log(str.match(/colou?r/g)); // Output: ["color", "colour"]

Zero or More (*)


let str = 'There are computer science & computational engineering';
let re = /comput\w*/g; // Matches "computer" and "computational"
console.log(str.match(re));// Output: ["computer", "computational"]

Examples

Greedy Quantifiers


    let regexp = /".+"/g;
    let str = 'The "Boy" and his "Friends" were here';
    console.log( str.match(regexp) ); // "Boy" and his "Friends"

Anchors

Anchors define positions within the input text where matches should occur.

  • ^: Matches the start of the input text
  • $: Matches the end of the input text
  • \b: Matches a word boundary
  • \B: Matches a non-word boundary

Examples


    let reg = /presentation$/g;
    let str = "Welcome to the presentation"; //matches

    let reg = /^javaScript/g;
    let str = "javaScript supports regular expressions"; //matches

Modifiers

Modifiers are optional flags that control the behavior of the regex pattern.

  • i: Case-insensitive matching
  • g: Global matching (matches all occurrences)
  • m: Multiline matching
  • s: Allows . to match newline characters
  • u: Enables full Unicode matching

Regular Expressions Methods

Regular expressions are used with the RegExp methods like test() and exec() and with the string methods match() , replace() , search() , and split() . These methods are explained in detail below with examples.

Examples

exec()


    console.log(/c/.exec("Coding is cool"));
    //['c', index: 10, input: 'Coding is cool', groups: undefined]
    console.log(/t/.exec("Coding is cool")); //null

test()


    console.log(/cool/.test("Coding is cool")); //true
    console.log(/col/.test("Coding is cool")); //false

match()


    console.log("Coding is cool".match(/Co/));
    // ['Co', index: 0, input: 'Coding is cool', groups: undefined]

search()


  console.log('Coding is cool'.search(/od/)); // 1
  console.log('Coding is cool'.search(/odo/)); // -1

replace()


  console.log('Coding is cool'.replace('Coding', 'JavaScript"));
  // JavaScript is cool

split()


  console.log('Hello world '.split(" "));// ['Hello', 'world!', '']

Applications

Regular Expressions are widely used in:

  • Text editors and IDEs
  • Programming languages (e.g., JavaScript, Python)
  • Web development (e.g., form validation, routing)
  • Data processing and analysis

Best Practices

  • Start simple and gradually build complexity
  • Test your expressions with various inputs
  • Use online regex testers and debuggers
  • Refer to documentation and cheat sheets

Conclusion

Regular Expressions are indispensable tools for text processing and manipulation. By understanding their syntax, common patterns, and best practices, you can leverage them effectively in various applications to search, match, and transform text data.
